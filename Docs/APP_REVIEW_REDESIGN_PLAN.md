# APP_REVIEW_REDESIGN_PLAN

## 1. 목적

이 문서는 Cryptory 앱의 Apple App Review 리젝 사유인 Guideline 3.1.5(iii) Cryptocurrency Exchanges 대응을 위해, 앱이 암호화폐 주문, 거래, 전송을 제공하는 앱으로 보이지 않도록 화면 구조와 문구, 진입 경로를 정리하는 설계 문서다.

이번 리디자인의 핵심은 기능 전체 제거가 아니라 App Store 제출 버전에서 주문, 매수, 매도, 전송, 입금, 출금, 주문 실행으로 해석될 수 있는 기능과 표현을 제거하거나 차단하는 것이다. 기존의 시세, 차트, 자산, 김프, 거래소 연동 기반 자산/포트폴리오 조회, 가격 알림, 관심 코인, 코인 상세 정보, 분석 정보, 뉴스/정보성 콘텐츠, 커뮤니티성 기능은 최대한 유지한다.

메인 탭에서는 기존 `주문` 탭만 제거하고 같은 위치에 `뉴스` 탭을 배치한다. 변경 후 메인 탭은 `시세 / 차트 / 뉴스 / 자산 / 김프`로 구성한다.

자산 탭과 거래소 연동은 제거하지 않는다. 다만 App Review 관점에서 거래 기능으로 오해되지 않도록 거래소 연동의 목적을 `읽기 전용 포트폴리오 조회`로 명확히 제한하고, 주문/출금 권한이 포함된 API Key 사용을 금지하는 안내와 방어선을 설계한다.

## 2. 현재 구조 요약

현재 메인 탭은 다음 5개로 추정된다.

| 현재 탭 | 현재 역할 추정 | App Review 관점 |
| --- | --- | --- |
| 시세 | 시장 가격, 코인 목록, 관심 코인, 가격 알림, 코인 상세 진입 | 정보성 기능으로 유지 가능 |
| 차트 | 캔들 차트, 기간 선택, 보조지표, 기술적 지표 | 정보성 기능으로 유지 가능하나 매수/매도 신호 표현 주의 필요 |
| 주문 | 주문, 매수, 매도, 호가, 체결, 거래 실행 진입점 | 리젝의 직접 원인 가능성이 높으므로 App Store 제출 버전에서 제거 필요 |
| 자산 | 거래소 연동, 포트폴리오, 보유 코인, 평가금액, 수익률 | 읽기 전용 자산 조회로 유지 가능하나 거래/전송 기능과 분리 필요 |
| 김프 | 국내외 시세 차이, 프리미엄 비율, 환율 기준, 거래소별 가격 비교 | 참고용 시장 데이터로 유지 가능하나 차익거래 유도 표현 금지 |

현재 구조에서 가장 큰 리스크는 `주문`이라는 메인 탭 자체와, 주문/매수/매도/체결/전송/입출금 화면으로 연결될 수 있는 라우팅이다. 메인 탭에서 주문 기능을 제거하더라도 딥링크, 푸시, 검색, 자산, 차트, 코인 상세 화면에서 주문 화면으로 진입할 수 있으면 App Review 대응이 불완전하다.

## 3. 변경 후 목표 구조

변경 후 메인 탭은 다음 5개로 구성한다.

| 순서 | 목표 탭 | 목표 역할 |
| --- | --- | --- |
| 1 | 시세 | 시장 가격, 코인 목록, 관심 코인, 가격 알림, 코인 상세 진입 |
| 2 | 차트 | 차트, 보조지표, 참고용 시장 신호, 최근 본 차트 |
| 3 | 뉴스 | 최신 뉴스, 주요 뉴스, 코인별 뉴스, 태그, 뉴스 상세, 원문 링크 |
| 4 | 자산 | 읽기 전용 거래소 연동 기반 포트폴리오 조회 |
| 5 | 김프 | 국내외 시세 차이, 프리미엄 비율, 환율 기준, 참고용 데이터 |

핵심 변경 사항은 기존 `주문` 탭 위치를 `뉴스` 탭으로 교체하는 것이다. `시세`, `차트`, `자산`, `김프`는 기존 기능을 유지하되 App Review에서 거래 앱으로 오해될 수 있는 CTA, 문구, 라우팅을 정리한다.

자산 탭은 반드시 유지한다. 단, 자산 탭의 성격은 `거래소 계정의 읽기 전용 자산/포트폴리오 조회`로 제한한다. 김프 탭은 거래 유도 기능이 아니라 국내외 시세 차이를 보여주는 참고용 정보 화면으로 유지한다.

## 4. 탭별 상세 설계

### 4-1. 시세 탭

#### 유지할 기능

시세 탭은 Cryptory의 핵심 시장 데이터 화면으로 유지한다. 다음 기능을 포함한다.

| 기능 | 설계 |
| --- | --- |
| 코인 목록 | 코인명, 심볼, 현재가, 등락률, 거래대금, 시가총액을 리스트로 표시 |
| 현재가 | 원화 또는 선택 기준 통화로 표시 |
| 등락률 | 상승/하락 색상으로 표시하되 거래 유도 CTA와 연결하지 않음 |
| 거래대금 | 시장 데이터 지표로 유지 가능 |
| 시가총액 | 코인별 시장 규모 정보로 표시 |
| 관심 코인 | 별 아이콘 또는 관심 목록으로 유지 |
| 가격 알림 | 특정 가격 도달 알림으로 유지하되 주문 실행과 연결하지 않음 |
| 급상승/급하락 코인 | 시장 변동성 정보로 제공 |
| 최근 검색 | 사용자가 최근 조회한 코인 목록 제공 |
| 코인 상세 진입 | 정보, 차트, 분석, 뉴스, 커뮤니티 섹션으로 연결 |

#### 추가 가능 기능

시세 탭에는 정보 탐색 기능을 강화하는 카드를 추가할 수 있다.

| 추가 기능 | 배치 | 설명 |
| --- | --- | --- |
| 인기 코인 카드 | 시세 탭 상단 또는 중단 | 조회수, 관심 등록, 뉴스 언급량 기준 인기 코인 표시 |
| 시장 요약 카드 | 시세 탭 상단 | 전체 시가총액, BTC 도미넌스, 시장 심리, 주요 지수 표시 |
| 코인별 정보 요약 | 코인 셀 또는 상세 진입 전 미리보기 | 시가총액 순위, 24시간 거래대금, 24시간 고가/저가 |
| 코인 상세 정보 섹션 | 코인 상세 내부 | CoinGecko형 정보, 설명, 공식 링크, 레딧 링크 |
| 코인 상세 커뮤니티 섹션 | 코인 상세 내부 | 의견, 투표, 게시글, 팔로우, 추천 멤버 |
| 코인별 뉴스 섹션 | 코인 상세 내부 | 해당 코인 태그가 붙은 최신 뉴스 |

#### 코인 상세 화면 설계

시세 탭에서 코인을 선택하면 코인 상세 화면으로 진입한다. 코인 상세는 다음 섹션 중심으로 설계한다.

| 섹션 | 내용 |
| --- | --- |
| 헤더 | 코인명, 심볼, 현재가, 등락률, 관심 등록, 가격 알림 |
| 정보 | 현재가, 시가총액, 유통 공급량, 최대 공급량, 총 공급량, 시장 데이터 |
| 차트 | 기간별 차트, 보조지표, 최근 가격 변동 |
| 분석 | 상승/중립/하락 중심의 참고용 시장 신호 |
| 뉴스 | 관련 코인 태그 기반 뉴스 |
| 커뮤니티 | 의견 게시글, 오른다/떨어진다 투표, 댓글, 좋아요, 팔로우 |

#### App Review 주의사항

시세 탭과 코인 상세 화면에서는 다음을 금지한다.

- `매수하기`, `매도하기`, `주문하기`, `거래하기` CTA 노출 금지
- 주문 화면, 매수 화면, 매도 화면, 호가 주문 화면 연결 금지
- 특정 뉴스나 가격 변동에서 즉시 거래를 유도하는 문구 금지
- 가격 알림 도착 후 주문 화면으로 이동하는 액션 금지

권장 문구:

- `시장 데이터를 확인하세요.`
- `관심 코인에 추가`
- `가격 알림 설정`
- `관련 뉴스 보기`
- `상세 정보 보기`

### 4-2. 차트 탭

#### 유지할 기능

차트 탭은 기존 차트 탐색 기능을 유지한다.

| 기능 | 설계 |
| --- | --- |
| 캔들 차트 | 코인별 가격 변동을 캔들 형태로 표시 |
| 기간 선택 | 1분, 5분, 15분, 30분, 1시간, 4시간, 1일, 1주 등 |
| 보조지표 | 이동평균, 거래량, RSI, MACD 등 기존 지원 범위 유지 |
| 관심 차트 | 사용자가 자주 보는 코인/마켓 차트 저장 |
| 최근 본 차트 | 최근 조회한 코인 차트 목록 |

#### 추가 가능 기능

참고 이미지의 기술적 분석 화면을 바탕으로 차트 탭 또는 코인 상세 차트 화면에 `시장 신호` 섹션을 추가할 수 있다.

| 기능 | 설계 |
| --- | --- |
| 시간 단위별 시장 신호 | 1분, 5분, 15분, 30분, 1시간, 2시간 등 선택 |
| 상승/중립/하락 요약 | `상승 신호`, `중립`, `하락 신호`로 표현 |
| 게이지 시각화 | 강한 하락 신호부터 강한 상승 신호까지 범위를 게이지로 표시 |
| 지표 카운트 | 상승 신호 수, 중립 지표 수, 하락 신호 수 표시 |
| 지표 설명 | 각 지표가 어떤 시장 데이터를 바탕으로 계산되는지 설명 |

#### 문구 가이드

기존 기술적 분석에서 거래 실행으로 오해될 수 있는 표현은 다음처럼 교체한다.

| 기존 표현 | 교체 표현 |
| --- | --- |
| 바이 | 상승 신호 |
| 셀 | 하락 신호 |
| 스트롱 바이 | 강한 상승 신호 |
| 스트롱 셀 | 강한 하락 신호 |
| 매수 신호 | 상승 관련 지표 |
| 매도 신호 | 하락 관련 지표 |
| 거래 신호 | 시장 참고 신호 |

#### 필수 고지

차트 탭과 분석 섹션에는 다음 고지를 노출한다.

> 차트와 분석 정보는 참고용 시장 데이터이며, 투자 조언이나 거래 신호가 아닙니다.

고지는 분석 화면 하단, 게이지 하단, 또는 상세 설명 펼침 영역에 표시한다. 사용자가 신호 정보를 거래 지시로 오해하지 않도록 `주문`, `매수`, `매도` 버튼과 결합하지 않는다.

### 4-3. 뉴스 탭

뉴스 탭은 기존 주문 탭을 대체하는 핵심 탭이다. 메인 탭의 세 번째 위치에 배치한다.

#### 기본 구조

| 영역 | 설계 |
| --- | --- |
| 상단 헤더 | `뉴스` 제목, 검색 아이콘, 키워드 필터 또는 코인 필터 |
| 제공사 표시 | `powered by Coinness` 또는 기존 뉴스 제공사 표시 |
| 카테고리 | 최신, 주요, 코인별, 시장, 규제, 이벤트 등 |
| 날짜 헤더 | `오늘, YYYY-MM-DD` 형식의 날짜 그룹 |
| 뉴스 리스트 | 시간 배지, 제목, 요약, 태그, 관련 코인 |
| 뉴스 상세 | 제목, 발행 시간, 제공사, 본문, 관련 태그, 원문 링크 |

#### 이미지 참고 UI 반영

참고 이미지의 뉴스 구조를 다음처럼 반영한다.

| UI 요소 | 설계 |
| --- | --- |
| 날짜 헤더 | 캘린더 아이콘과 함께 `오늘, 2026-04-30`처럼 표시 |
| 시간 배지 | 각 뉴스 좌측에 `12:46`, `12:39` 형태 배지 표시 |
| 강조 제목 | 주요 뉴스는 강조 색상 또는 굵은 제목으로 표시 |
| 요약 본문 | 2~4줄 요약을 제공하고 긴 본문은 상세에서 표시 |
| 태그 | `#BTC`, `#ETH`, `#SOL` 등 관련 코인 태그 표시 |
| 스크롤 리스트 | 최신 뉴스가 시간순으로 이어지는 세로 리스트 |

#### 포함할 기능

뉴스 탭에는 다음 기능을 포함한다.

| 기능 | 설계 |
| --- | --- |
| 최신 뉴스 | 시간순 뉴스 피드 |
| 주요 뉴스 | 중요도 또는 제공사 플래그 기반 강조 |
| 코인별 뉴스 | 코인 태그 필터로 특정 코인 뉴스 모아보기 |
| 관련 코인 태그 | 뉴스에 연결된 코인 태그 표시 |
| 날짜별 뉴스 그룹 | 오늘, 어제, 날짜별 그룹 |
| 시간 배지 | 뉴스 발행 시간 표시 |
| 뉴스 요약 | 리스트에서 요약 표시 |
| 뉴스 상세 | 본문 전체, 출처, 원문 링크 |
| 원문 링크 | 외부 브라우저 또는 인앱 웹뷰로 원문 이동 |
| 제공사 표시 | `powered by Coinness` 또는 실제 제공사명 표시 |
| 캘린더/이벤트 통합 | 기존 캘린더/이벤트 기능이 있다면 뉴스 탭 내부 섹션으로 통합 가능 |

#### 추가 가능 기능

| 기능 | 설명 |
| --- | --- |
| 뉴스 북마크 | 사용자가 나중에 읽을 뉴스를 저장 |
| 관련 코인 바로가기 | 뉴스 태그를 눌러 코인 상세 정보로 이동 |
| 관련 코인의 시세/차트 연결 | 코인 상세 또는 차트 화면으로 연결하되 주문 화면 연결 금지 |
| 뉴스 키워드 필터 | ETF, 규제, 거래소, 해킹, 상장 등 키워드 필터 |
| 코인별 뉴스 모아보기 | 코인 상세 내부 뉴스 섹션과 동일 데이터 사용 |

#### 금지 사항

뉴스 탭과 뉴스 상세에서는 다음을 금지한다.

- 뉴스에서 `매수`, `매도`, `주문`, `거래하기` CTA 연결 금지

자산 탭 상단, 거래소 연동 화면, API Key 등록 화면에 다음 문구를 표시한다.

> 거래소 연동은 읽기 전용 자산 조회 목적으로만 사용됩니다. Cryptory는 암호화폐 매수·매도·전송·입금·출금·주문 실행 기능을 제공하지 않습니다.

#### API Key 등록 화면 설계

API Key 등록 화면은 App Review 대응에서 매우 중요하다. 거래소 연동을 유지하되 거래 기능이 없다는 점을 명확히 해야 한다.

| 항목 | 설계 |
| --- | --- |
| 화면 제목 | `읽기 전용 거래소 연동` |
| 안내 문구 | `자산 조회 전용 API Key만 등록할 수 있습니다.` |
| 권한 안내 | `주문 권한 또는 출금 권한이 포함된 API Key는 등록하지 마세요.` |
| 권한 체크 | 등록 전 읽기 전용 권한 확인 체크박스 표시 |
| 보안 안내 | API Key는 포트폴리오 조회 목적으로만 사용된다는 설명 |
| 등록 버튼 | `읽기 전용 API Key 연결` |

필수 체크박스:

- `이 API Key가 읽기 전용 권한임을 확인했습니다.`
- `주문/출금 권한이 포함된 API Key를 등록하지 않겠습니다.`

추가 방어선:

- 거래소별 API Key 권한 검증이 가능하면 주문/출금 권한 포함 여부를 감지한다.
- 권한 검증이 불가능한 거래소는 사용자가 읽기 전용 권한임을 직접 확인해야 등록 가능하게 한다.
- App Store mode에서는 주문/출금 권한이 감지된 API Key를 거부하거나 자산 조회 외 기능을 모두 차단한다.

## 6. 거래소 연동 유지 설계

거래소 연동은 제거하지 않는다. App Review 대응에서 중요한 것은 거래소 연동의 목적과 권한을 `읽기 전용 자산 조회`로 제한하는 것이다.

### 설계 원칙

| 원칙 | 설명 |
| --- | --- |
| 읽기 전용 | 거래소 API Key는 잔고, 보유 자산, 평가금액 조회에만 사용 |
| 주문 권한 금지 | 주문, 매수, 매도 권한이 포함된 API Key 등록 금지 |
| 출금 권한 금지 | 출금, 전송 권한이 포함된 API Key 등록 금지 |
| API 분리 | 자산 조회 API와 거래 실행 API를 클라이언트/서버 양쪽에서 분리 |
| App Store mode 차단 | App Store 제출 빌드에서는 거래성 API 호출을 클라이언트와 서버에서 모두 차단 |

### 클라이언트 방어선

| 영역 | 설계 |
| --- | --- |
| API Key 등록 | 읽기 전용 권한 안내와 체크박스 필수 |
| 거래 기능 플래그 | `isOrderEnabled`, `isTradingEnabled`, `isTransferEnabled` false |
| UI 차단 | 매수/매도/전송/입출금 CTA 렌더링 금지 |
| Repository 차단 | 주문/출금/입금 API 호출 메서드가 App Store mode에서 실행되지 않도록 보호 |
| 라우팅 차단 | 거래성 화면으로 이동하는 route를 무시하거나 정보 화면으로 대체 |

### 서버 방어선

| 영역 | 설계 |
| --- | --- |
| App Store mode | 서버 feature flag 또는 build channel 기준으로 거래성 API 차단 |
| 주문 API | App Store mode에서 403 또는 비활성화 응답 |
| 전송/출금 API | App Store mode에서 403 또는 비활성화 응답 |
| API Key 권한 검증 | 가능하면 주문/출금 권한 포함 API Key 거부 |
| 감사 로그 | 차단된 거래성 API 호출을 로그로 남겨 검증 가능하게 함 |

### 사용자 안내 문구

거래소 연동 화면에는 다음 메시지를 명확히 노출한다.

> Cryptory의 거래소 연동은 읽기 전용 포트폴리오 조회를 위한 기능입니다. Cryptory는 암호화폐 매수, 매도, 주문 실행, 전송, 입금, 출금 기능을 제공하지 않습니다.

## 9. Feature Flag 설계

구현 전 다음 feature flag를 설계한다. App Store 제출 빌드에서는 거래성 기능이 확실히 false가 되도록 빌드 설정, 원격 설정, 서버 설정이 일관되어야 한다.

| Flag | App Store 제출 값 | 설명 |
| --- | --- | --- |
| isMarketEnabled | true | 시세 탭 활성화 |
| isChartEnabled | true | 차트 탭 활성화 |
| isNewsEnabled | true | 뉴스 탭 활성화 |
| isReadOnlyPortfolioEnabled | true | 읽기 전용 자산/포트폴리오 활성화 |
| isKimchiPremiumEnabled | true | 김프 탭 활성화 |
| isCommunityContentEnabled | true | 커뮤니티/의견형 콘텐츠 활성화 |
| isOrderEnabled | false | 주문 기능 비활성화 |
| isTradingEnabled | false | 매수/매도/거래 실행 비활성화 |
| isTransferEnabled | false | 전송/송금 비활성화 |
| isDepositWithdrawEnabled | false | 입금/출금 비활성화 |
| isWalletEnabled | false | 지갑/지갑주소 기능 비활성화 |
| isPrivateExchangeTradingAPIEnabled | false | 거래소 private trading API 비활성화 |

### Flag 적용 원칙

- App Store 빌드에서 주문/거래/전송/입출금/지갑 기능은 false로 고정한다.
- 자산/거래소 연동은 true로 유지하되 읽기 전용 조회 기능만 허용한다.
- 원격 설정 오류로 거래 기능이 켜지지 않도록 App Store build channel에서는 클라이언트 하드가드가 필요하다.
- 서버도 App Store mode에서 거래성 API를 차단해야 한다.
- UI, 라우팅, API 호출, 딥링크, 푸시 이동 모두 같은 flag 정책을 사용해야 한다.

## 11. 서버 영향 범위 후보

서버는 뉴스, 시장 데이터, 포트폴리오 조회 기능은 유지/강화하고, 주문/거래/전송 API는 App Store mode에서 차단하는 방향으로 설계한다.

| 후보 | 목표 |
| --- | --- |
| news routes/service | 뉴스 탭 데이터 제공, 코인 태그, 날짜 그룹, 원문 링크 지원 |
| portfolio routes/service | 읽기 전용 포트폴리오 조회 유지 |
| exchange connection routes/service | API Key 등록, 권한 안내, 가능 시 권한 검증 |
| market data routes/service | 시세, 현재가, 등락률, 거래대금, 시가총액 유지 |
| kimchi premium routes/service | 국내외 가격 비교, 환율 기준, 프리미엄 계산 유지 |
| coin info routes/service | CoinGecko형 코인 상세 정보, 공급량, 설명, 링크 제공 |
| community routes/service | 게시글, 투표, 댓글, 좋아요, 팔로우 제공 |
| order/trading routes/service | App Store mode에서 비활성화 또는 403 |
| transfer/withdraw/deposit routes/service | App Store mode에서 비활성화 또는 403 |
| compliance middleware | App Store mode 요청에서 거래성 API를 공통 차단 |
| feature flag config | 클라이언트/서버 동일 정책 유지 |
| audit logging | 차단된 거래성 API 호출 기록 |

### 서버 응답 설계

거래성 API가 App Store mode에서 호출될 경우 다음처럼 명확히 차단한다.

```json
{
  "code": "FEATURE_DISABLED_FOR_APP_STORE",
  "message": "Trading, transfer, deposit, and withdrawal features are not available in this app version."
}
```

클라이언트는 이 응답을 받더라도 거래 화면을 열지 않고 정보성 화면 또는 오류 안내만 표시해야 한다.

## 12. 단계별 구현 계획

현재 단계에서는 구현하지 않는다. 다음 단계에서 구현자가 따라갈 수 있도록 단계별 계획만 정의한다.

1. 현재 탭/라우팅 구조 조사
   - MainTabBarController, MainTabCoordinator, RootTab enum, TabItem model 확인
   - 현재 `주문` 탭이 어떤 화면과 coordinator에 연결되는지 파악

2. 주문 탭 의존성 조사
   - OrderViewController, OrderCoordinator, OrderRepository, TradingUseCase 확인
   - 주문 화면을 참조하는 딥링크, 푸시, 검색, 자산, 차트, 코인 상세 경로 조사

3. 뉴스 화면 기존 구현 여부 조사
   - NewsViewController 또는 News module 존재 여부 확인
   - 기존 뉴스 API, Coinness 연동, 캘린더/이벤트 기능 존재 여부 확인

4. 주문 탭에서 뉴스 탭으로 교체 설계 확정
   - 탭 index 유지 여부 확인
   - 기존 세 번째 탭 위치에 News root 연결
   - analytics, accessibility label, icon, localization key 수정 범위 확정

5. 주문/거래/전송 진입 경로 제거
   - CTA 제거
   - 딥링크 차단
   - 푸시 라우팅 차단
   - 검색 결과 제외
   - 자산/차트/코인 상세에서 주문 화면 연결 제거

6. 자산 탭 읽기 전용 고지 추가
   - 자산 탭 상단 또는 안내 영역에 필수 고지 표시
   - API Key 등록 화면에 읽기 전용 체크박스 추가
   - 주문/출금 권한 포함 API Key 금지 안내 추가

7. 뉴스 탭 UI 구성
   - 날짜 헤더, 시간 배지, 제목, 요약, 태그, 제공사 표시
   - 뉴스 상세, 원문 링크, 코인 태그 연결
   - 뉴스에서 주문 CTA가 없는지 확인

8. 각 탭별 추가 기능 배치
   - 코인 상세 정보 강화
   - 차트/분석의 문구를 시장 신호 중심으로 변경
   - 커뮤니티 기능을 코인 상세 또는 뉴스 탭 의견 섹션에 배치
   - 김프 설명 카드와 계산 기준 안내 추가

9. App Review 문구 점검
   - 주문, 매수, 매도, 체결, 전송, 입금, 출금, 지갑, 차익거래 등 문구 검색
   - 유지 가능한 시장 데이터 용어와 제거해야 할 거래성 용어를 분리

10. 테스트 체크리스트 수행
    - UI, 라우팅, 딥링크, 푸시, API 호출, 문구, App Store mode flag 검증
